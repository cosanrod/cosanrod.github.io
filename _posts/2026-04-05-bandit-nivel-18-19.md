### 🧠 Explicación

#### 🔍 ¿Qué está pasando?

El archivo:

```bash 
.bashrc
```

es un script que se ejecuta automáticamente al iniciar una sesión interactiva.

En este caso:

1. Te conectas por SSH
2. Se carga el entorno del usuario
3. `.bashrc` ejecuta un `exit`
4. La sesión se cierra inmediatamente

👉 No te deja escribir comandos.

---

### 🚀 La solución: ejecución remota

SSH permite ejecutar comandos directamente:

```bash 
ssh usuario@host "comando"
```

Esto hace que:

* el comando se ejecute inmediatamente
* se imprima el resultado
* la sesión se cierre después

💡 Y lo más importante:

> evita (o reduce el impacto de) la ejecución interactiva del `.bashrc`

---

### 💡 Tip

Este truco es muy útil cuando:

* hay restricciones en el shell
* scripts bloquean la sesión
* necesitas ejecutar algo rápido sin interactuar
