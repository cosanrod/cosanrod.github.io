### 💻 Solución paso a paso

Primero, dentro del servidor encontramos la clave privada (por ejemplo `sshkey.private`) y copiamos su contenido.

Luego, en nuestra máquina local:

Creamos un archivo para guardar la clave:

```bash id="p3m9zs"
touch llave
```

Editamos el archivo y pegamos la clave:

```bash id="t6x2rc"
nano llave
```

Guardamos y ajustamos permisos:

```bash id="m8c3qy"
chmod 600 llave
```

Finalmente, nos conectamos usando la clave privada:

```bash id="x1k4rb"
ssh -i llave bandit14@bandit.labs.overthewire.org -p 2220
```

Salida:

```bash id="z7n5lw"
bandit14@bandit:~$
```

### 🧠 Explicación

Este nivel introduce autenticación con **llaves SSH**:

* `-i` → permite especificar una clave privada
* `chmod 600` → restringe permisos (solo el dueño puede leer/escribir)

⚠️ Importante:
SSH no permite usar claves con permisos abiertos por seguridad.

Flujo:

1. Obtener clave privada
2. Guardarla en un archivo
3. Ajustar permisos
4. Usarla para autenticarse

---

### 💡 Tip

Las llaves SSH son muy comunes en:

* servidores reales
* despliegues automáticos
* pentesting (cuando encuentras claves expuestas 👀)
